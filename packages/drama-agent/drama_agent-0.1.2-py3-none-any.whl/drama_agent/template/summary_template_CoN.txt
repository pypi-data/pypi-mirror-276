“剧本: {script}”
您将生成越来越简洁、实体密集的上述文章摘要。

重复以下2个步骤5次。

第1步：从文章中识别1-3个有信息量的实体（用“;”分隔），这些实体在先前生成的摘要中缺失。
第2步：编写一个新的、更密集的摘要，长度相同，涵盖以前摘要中的每个实体和细节以及缺失的实体。

缺失的实体是：
- 与主要故事相关。
- 具体：描述性但简洁（5个词或更少）。
- 新颖：不在以前的摘要中。
- 忠实：出现在文章中。
- 任何地方：出现在文章的任何位置。

指南：
- 第一个摘要应该很长（4-5句，大约80个字），但非常不具体，除了标记为缺失的实体以外，包含很少的信息。使用过于冗长的语言和填充词（例如“本文讨论”）以达到大约80个字。
- 让每个词都有意义：重新编写以前的摘要以改进流畅度，并为额外的实体腾出空间。
- 利用融合、压缩和删除像“文章讨论”这样不具信息价值的短语来节省空间。
- 摘要应变得非常密集和简洁，但自成一体，例如，可以在不阅读文章的情况下轻松理解。
- 缺失的实体可以出现在新摘要的任何位置。
- 绝不要从以前的摘要中删除实体。如果无法腾出空间，请添加较少的新实体。

请记住，每个摘要都使用相同数量的词。以JSON格式回答。JSON应该是一个包含5个字典的列表，其键是"Missing_Entities"和"Denser_Summary"。
